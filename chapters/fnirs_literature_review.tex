\section{Abstract}

This report compares two image classification models trained on the same coffee leaf
disease dataset. The objective is to classify images of coffee leaves into three
classes: healthy leaves, coffee leaf rust, and Phoma disease. The models evaluated are
MobileNetV2 transfer learning and a custom Simple CNN. Both models use the same
training, validation, and testing folders, batch size, and input image size
(224 \(\times\) 224).

The Simple CNN reached higher test accuracy (95.88\%) than MobileNetV2 (94.43\%).
However, MobileNetV2 was substantially faster to train in the main frozen-base run
(about 16 minutes), while the Simple CNN required about 82 minutes.

\section{Introduction}

Coffee leaf diseases can reduce crop productivity and quality. Computer-vision models
can support automatic disease detection by learning from labeled images of diseased and
healthy leaves. In this project, the goal is not only to train accurate classifiers,
but also to compare two different approaches and analyze practical trade-offs.

The first model uses transfer learning with MobileNetV2, where only a lightweight
classification head is trained and the pre-trained feature extractor remains frozen.
The second model is a custom CNN trained from scratch, where all convolution filters are
learned directly from the project data. This comparison is relevant because the best
accuracy is not always the best deployment choice.

\section{Dataset Description}

The dataset used in the notebooks is ``A Machine Learning Dataset for Classification of
Common Coffee Leaf Diseases in Uganda'' published on Mendeley Data with
DOI: 10.17632/k36wnd6knb.1 \cite{Chelangat2025}. The source describes labeled coffee
leaf images collected under varied farm lighting conditions in Uganda, with three
classes: healthy leaves, leaf rust, and Phoma disease \cite{Chelangat2025}.

The source indicates 3,312 images, while the notebook workflow detected a slightly
different local file count before cleanup. After corrupted-image cleansing, the run used
2,252 training images, 483 validation images, and 485 test images, totaling

\section{Preprocessing and Experimental Setup}

Both models were trained with the same data split configuration. Images were loaded from
folders, resized to 224 \(\times\) 224, batched with batch size 32, and encoded as
categorical labels. The training set used augmentation (random rotation, width shift,
height shift, horizontal flip, and zoom). Validation and test sets were rescaled only.

Both models were compiled with categorical cross-entropy loss and Adam optimizer with an
initial learning rate of 0.001. Training used EarlyStopping, ReduceLROnPlateau, and
ModelCheckpoint callbacks. These callbacks are documented in Keras as strategies for
automatic early termination and learning-rate reduction under plateau behavior
\cite{KerasEarlyStopping,KerasReduceLROnPlateau}. Classification reports were generated
with scikit-learn classification\_report \cite{ScikitClassificationReport}.

One methodology issue should be corrected in a future run. Keras recommends MobileNetV2
preprocessing through preprocess\_input (mapping to \([-1, 1]\)) rather than plain
rescaling to \([0, 1]\) \cite{KerasMobileNet}. The achieved score is still high, but
this preprocessing mismatch should be fixed for a cleaner transfer-learning experiment.

\section{Model 1: MobileNetV2 Transfer Learning}

Model 1 uses MobileNetV2 as a frozen feature extractor with a custom classifier head.
MobileNetV2 is based on inverted residuals and linear bottlenecks \cite{Sandler2018},
and Keras supports ImageNet pre-trained variants for transfer learning
\cite{KerasMobileNet,KerasTransferLearning}.

The notebook loads MobileNetV2 with include\_top=False and ImageNet weights, freezes the
base model, and adds global average pooling, dropout (0.3), and a 3-class softmax
classifier.

\section{Model 2: Simple CNN}

Model 2 is a custom convolutional network trained from scratch without ImageNet
pre-trained weights. The architecture includes Conv2D, BatchNormalization,
MaxPooling2D, and Dropout blocks, followed by dense classification layers with a
3-class softmax output.

Compared with transfer learning, this approach trains all model parameters on the project
dataset, which increases training time but can improve adaptation to
dataset-specific visual patterns.

\section{Model Comparison and Discussion}

The Simple CNN produced the highest overall test accuracy in this run: 95.88\% versus
94.43\% for MobileNetV2, a difference of 1.45 percentage points. It also improved Phoma
recall from 0.87 to 0.92, reducing false negatives for that class.

At the same time, MobileNetV2 remained much faster to train. This follows directly from
the number of trainable parameters in the main run: 3,843 for MobileNetV2 versus
1,442,851 for the Simple CNN. Therefore, model selection depends on deployment
requirements, including acceptable training time and retraining frequency.

Both models effectively classified healthy leaves, as reflected in their high true
positive and true negative values for that class. However, MobileNetV2 classified all
healthy test samples correctly while misclassifying 18 images labelled as Phoma into the
leaf rust category. In contrast, the Simple CNN made some mistakes on healthy samples
but reduced the Phoma-to-rust confusion, misclassifying only 12 Phoma samples as leaf
rust. Thus, the Simple CNN better distinguished the two disease classes in this run.

Interpretation also requires caution. The dataset source notes use of augmentation for
sample diversity \cite{Chelangat2025}. If similar augmented variants appear across split
boundaries, reported accuracy may be optimistic due to leakage. Related plant-disease
literature also emphasizes robust evaluation design and domain constraints
\cite{Dhaka2021,CABI2026,RevistaCultivar2024}.

\section{Limitations and Improvements}

The assignment goal is fulfilled because both models were trained and tested on the same
coffee leaf dataset. To improve scientific rigor, the following steps are recommended:

\begin{enumerate}
\item Apply official MobileNetV2 preprocessing with preprocess\_input instead of
rescale=1/255 and rerun training and testing \cite{KerasMobileNet}.
\item Evaluate the fine-tuned MobileNetV2 block separately on the test set and report
its final metrics \cite{KerasTransferLearning}.
\item Repeat experiments across multiple random seeds and report average accuracy and
average F1-score.
\item Strengthen split design to prevent leakage across train, validation, and test
sets.
\item Validate model robustness on an additional external coffee-leaf dataset.
\item Measure inference latency and memory consumption on the target device.
\item Add explainability analysis, for example Grad-CAM, to verify that models focus on
disease regions rather than background artifacts.
\end{enumerate}
