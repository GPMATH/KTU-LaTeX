\section{Abstract}

In this report, two image classification models trained on the same coffee leaf disease
dataset were compared. The objective of this study was to classify images of coffee leaves
into one of three classes -- healthy leaves, coffee leaf rust, and Phoma disease. The
models tested in this experiment were a MobileNetV2 transfer learning model and a Simple
CNN. Both models used the same training, validation, and testing folders, the same batch
size, and input images of 224 x 224 pixels. The three output classes for both
models were the same as well. The Simple CNN model performed better in terms of test
accuracy, achieving 95.88\%, compared to 94.43\% in the case of the MobileNetV2 model.
However, in terms of the training time, the Simple CNN model needed 82 minutes for
training while MobileNetV2 needed only 16 minutes for the frozen-base model run.

\section{Introduction}

Coffee leaf diseases may decrease crop productivity and quality. Machine vision-based
computer models can assist farmers with automated disease detection by learning from
labeled images of healthy and diseased leaves. In the current project, the task was not only to build a
model that performed classification accurately, but also to compare two methods and
analyze the differences between the chosen models. Such analysis was useful since the
model with the best accuracy was not necessarily the most efficient one to deploy. In this
paper, the criteria for comparison included such parameters as accuracy, loss, performance
per class, the number of model parameters, preprocessing, limitations and possible
problems with the dataset.

Two models discussed in this experiment belonged to two widely-used types of image
classification techniques. The first one used transfer learning based on pre-trained
MobileNetV2. Only a classification head was trained while the remaining feature
extraction part of the model was kept frozen. The second approach used a custom CNN
architecture which learned convolutional filters during the training process using the
dataset. Both of these models were applied depending on the conditions: transfer learning
in case there was a lack of labeled data and vice versa.

\section{Dataset Description}

The external dataset used in this experiment was ``A Machine Learning Dataset for
Classification of Common Coffee Leaf Diseases in Uganda.'' The dataset was posted on
Mendeley Data with DOI: 10.17632/k36wnd6knb.1 \cite{Chelangat2025}. According to the source page, the
dataset contained a set of labeled images of coffee leaf diseases collected under various
lighting conditions on farms in Uganda. These images were classified into three classes:
healthy leaves, leaf rust, and Phoma disease \cite{Chelangat2025}. Images were stored as JPEG files of
256x256 pixels, whereas the experiment loaded the images at 224x224 pixels prior to the
training \cite{Chelangat2025}.

An important point about reproducibility of this dataset was noted. According to
the description, the Mendeley source provided 3,312 images, but the local dataset contained
a slightly different number of files before cleanup. Corrupted files were then cleaned,
and 2,252 training images, 483 validation images, and 485 test images were used
for the model training in total of 3,220 images. In this report, the external
source was used for documenting the dataset origin, and model results were obtained with
the exact numbers provided by the experimental outputs.

\section{Preprocessing and Experimental Setup}

The images were preprocessed and prepared for training in the same way for both models.
The images were read from folders and scaled to 224 x 224 pixels. They were batched with
batch size 32 and transformed to categorical labels. The training set was augmented with
random rotations, width shifts, height shifts, horizontal flips, and zooming. Validation
and test data were not augmented; they were only rescaled.

Models were compiled with categorical cross-entropy as a loss function and Adam optimizer
with the initial learning rate of 0.001. The early stopping strategy, reduce learning
rate on plateau, and save best model were used during model training. As documented by
Keras, EarlyStopping was a callback that stopped model training when no improvement in the
monitored quantity was observed. Similarly, ReduceLROnPlateau was a callback that
decreased learning rate upon a lack of improvement during training \cite{KerasEarlyStopping,KerasReduceLROnPlateau}. Finally,
the classification reports were generated with scikit-learn using its classification\_report
function \cite{ScikitClassificationReport}.

A methodology limitation was related to the MobileNetV2 preprocessing pipeline. According
to the MobileNetV2 documentation in Keras, rescaling to [0..1] interval was not recommended. The
documentation recommended using the MobileNetV2 preprocessing function to rescale inputs
to [-1..1] interval. Even though the model achieved very good results, its preprocessing
pipeline was not fully aligned with the recommended transfer-learning preprocessing method.

\section{Model 1: MobileNetV2 Transfer Learning}

Model 1 applied MobileNetV2 for feature extraction with transfer learning. Specifically,
MobileNetV2 was proposed as an efficient mobile network with inverted residuals and
linear bottlenecks \cite{Sandler2018}. Similarly, the Keras application guide mentioned that MobileNetV2
applied inverted residual blocks with bottlenecks and supported ImageNet pre-trained
weights \cite{KerasMobileNet}.

The MobileNetV2 base was loaded with ImageNet weights, the original top
classifier was removed, and the base was frozen. The new classification head contained global average
pooling, a 0.3 rate dropout, and a dense softmax classifier with three output classes.
Thus, the workflow matched the typical transfer learning pipeline as follows: loading
pre-trained base model, freezing the base model, creating and stacking the new model on top, and
then training the classifier on the target data \cite{KerasTransferLearning}.

\section{Model 2: Simple CNN}

Model 2 was a custom convolutional neural network trained from scratch. In contrast to
Model 1, Model 2 did not use pre-trained ImageNet weights. The architecture consisted of
several convolutional blocks with Conv2D layers, batch normalization, max pooling, and
dropout. The last classifier contained dense layers with three class softmax outputs.
Therefore, this design enabled Model 2 to extract features from the coffee leaf images
directly. Still, the training time was longer compared to MobileNetV2 since
all trainable filters needed to be learned from scratch using the project dataset.

\section{Model Comparison and Discussion}

The Simple CNN yielded the highest numerical score in this run, achieving a test accuracy
of 95.88\%, compared to 94.43\% by MobileNetV2. The gap between the two was 1.45
percentage points. Furthermore, the Simple CNN model also improved the recall value of
the Phoma label from 0.87 to 0.92. This was an improvement of 0.05, which indicated that
Simple CNN identified fewer false-negative Phoma instances than MobileNetV2. This was
relevant since disease identification aimed to minimize false negatives.

However, Simple CNN was not automatically the superior model. The
Simple CNN required approximately 82 minutes for training, compared to around 16 minutes
for MobileNetV2. This was because only 3,843 parameters were trainable in the main run of
MobileNetV2, compared to 1.44 million for the Simple CNN model. Training time was an
important criterion when quick model testing was crucial, thus making MobileNetV2 more
efficient.

The confusion matrix results revealed that both models could effectively classify healthy
leaves, as seen by their high values in the True Positive and True Negative cells.
However, MobileNetV2 could identify all healthy test samples but classified 18 images
labeled as Phoma into the leaf rust category. In contrast, the Simple CNN made mistakes
when identifying healthy samples but minimized Phoma-to-rust confusion by classifying 12
Phoma samples as leaf rust instead. Thus, Simple CNN seemed to better distinguish the two
classes in this run.

Finally, care was needed in interpreting the result since the dataset source mentioned
that augmentation techniques were utilized to improve sample diversity \cite{Chelangat2025}. Augmented
versions of the same leaves might have been included in each split, leading to an
overestimated measurement of accuracy. While random training, validation, and test folders
were generated, there was no evidence that the data were split in a manner
that ensured each sample group contained similar augmented variants.

\section{Limitations and Improvements}

The requirement for the assignment was fulfilled since both models were trained and tested
on the same dataset with coffee leaves. Some limitations affected the scientific nature of
the report:

\begin{enumerate}
\item The official MobileNetV2 preprocessing pipeline was not applied, since the current
rescale = 1/255 was used instead of the preprocess\_input() function.

\item The fine-tuned MobileNetV2 model was not evaluated separately on the test set. Although the
fine-tuning block of the MobileNetV2 model was included, there was no evaluation of its results.

\item The code was not run using several random seeds, so the average accuracy and average
F1-score were not reported.

\item Possible data leakage was not fully excluded when splitting the datasets into training,
validation, and test sets. Augmented images or duplicates could have appeared in other sets
besides the test one.

\item The models were not evaluated on an additional dataset or with images collected
during the project.

\item The inference time per image and memory consumption on the targeted device were not estimated.
Training time alone could not be the determinant for selecting a particular model.

\item An explanation technique such as Grad-CAM was not added to confirm that the models learned
features from disease regions rather than background artifacts.
\end{enumerate}
